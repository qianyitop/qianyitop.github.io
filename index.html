渗透测试技巧：巧用目录爆破发现隐藏后台

在渗透测试的“信息收集”阶段，目录与文件爆破是最快出成果的手段之一。
下面分享一条常被忽视却屡试不爽的小技巧：“先字典、后规则、再人工”。

  1. 先字典
    别一上来就甩 20 万条大字典。先用 3～5 份“业务敏感小字典”跑一遍，比如：

    • 常见后台：admin、manager、console、system
    • 语言特征：phpmyadmin、api-docs、swagger-ui
    • 云原生路径：actuator、metrics、env
      小字典跑完往往就能命中 60 % 的隐藏入口，省时省力。

  2. 后规则
    当小字典无收获时，启动“规则引擎”——用 ffuf / feroxbuster 的 -fw 自动过滤 404 页面宽度，再配合自定义正则剔除“假 200”页面（如“系统繁忙，请稍后再试”）。

    这样可把 8 万条字典的误报率压到 5 % 以下。

  3. 再人工
    爆破结果出来后，别急着丢给漏洞扫描器。先人工快速浏览响应体，重点看三类特征：

    • 响应里出现 login、password、token 关键字 → 登录口
    • Content-Length 异常小（<100 B）却返回 200 → 可能是未授权接口
    • 响应头出现 X-Accel-Redirect、Location: internal- → Nginx 内部跳转，大概率藏着“换皮后台”

Bonus 技巧
如果目标站使用 CDN，把 Host 头改成内网 IP 再跑一次字典，经常能直接命中源站隐藏目录，从而绕过 CDN 缓存与 WAF 拦截。

一句话总结：
“小字典快速验证 → 规则降噪 → 人工二次筛查”，三步下来，隐藏后台无处遁形。祝各位早日拿到高危及高危以上的“小可爱”。
