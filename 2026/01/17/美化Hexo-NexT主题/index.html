给 Hexo + NexT 做一次轻量美化（含配置清单）

发表于 2026-01-17
本文字数： 1.1k 阅读时长 ≈ 2 分钟

这篇文章记录我对本站（Hexo + NexT / Gemini scheme）做的一次「轻量但耐看」的美化过程：尽量不改主题源码，主要通过 source/_data/next.yml + 自定义 Stylus 覆盖变量与样式来实现。这样后续升级主题也更省心。

目标

  • 让页面更“干净”：卡片化、圆角、阴影更克制
  • 让阅读更舒服：更合适的字号/行距/字体栈、阅读进度条
  • 让交互更顺手：代码块一键复制、图片点击放大
  • 兼容深色模式：暗色下也保持对比度与层次

1. 找到正确的配置入口

本站主题是 NexT，并且支持用 source/_data/next.yml 合并配置（不需要直接改主题目录下的 _config.yml）。

  • 站点配置：e:\Sec\blog\_config.yml
  • 主题默认配置：themes\hexo-theme-next\_config.yml
  • NexT 会把 source/_data/next.yml 合并进主题配置（可追溯到 config.js）

2. 用 next.yml 开启“常用美化功能”

新增并写入：source\_data\next.yml

我主要做了这些开关：

  • 开启阅读进度条（顶部细条）
  • 开启代码块复制按钮（mac 风格）
  • 开启图片点击放大（mediumzoom）
  • 开启 pangu（中英文自动加空格）
  • TOC 支持自动换行（标题长的时候更友好）

3. 通过变量覆盖做统一的视觉基调

NexT 的 CSS 使用 Stylus 变量组织，这意味着我们可以通过注入一个 variables.styl 来覆盖主题默认值。

新增并写入：source\_data\variables.styl

这一步我做了几件事：

  • 背景色与卡片色：从“纯白/纯灰”改为更柔和的浅色底，并给暗色做了更深的底色
  • 统一圆角：提升到 14px，让卡片更现代
  • 阴影更克制：用更柔和的扩散阴影来做层次
  • 字体栈更稳：不依赖远程字体，优先使用系统字体（Windows/macOS/移动端都有不错效果）

4. 用 styles.styl 做“少量但关键”的细节增强

新增并写入：source\_data\styles.styl

主要增强点：

  • 背景做轻微渐变与光晕（固定背景，滚动时更有层次）
  • 卡片加半透明 + 模糊（backdrop-filter），并补了暗色边框
  • 卡片 hover 轻微上浮（更“可点击”）
  • 图片圆角与阴影，提升观感
  • 代码块圆角与裁切，整体更统一

5. 本次改动清单

  • 新增主题合并配置：source\_data\next.yml
  • 新增变量覆盖：source\_data\variables.styl
  • 新增样式增强：source\_data\styles.styl

如果你也在用 NexT，建议优先走 source/_data 这条“注入式”方案：改动更聚焦，也更利于维护。
